1. Introduction

Le Mod Loader de Rock Star Career te permet d'injecter du code JavaScript pendant une partie pour ajouter des contenus, modifier les valeurs du jeu, ou créer des mécaniques entièrement nouvelles.

Tout se fait via l'objet global RSC_MOD exposé par le jeu. Tes mods sont des fichiers .js simples que tu importes depuis le menu 🔧 Mods en jeu.

💡 Tu peux aussi coller et exécuter du code directement dans l'éditeur intégré au menu Mods, sans créer de fichier.
⚠️ Les mods s'exécutent dans le contexte de la page. Un mod mal écrit peut crasher ta partie. Pense à sauvegarder avant d'en charger un nouveau.

2. Structure d'un mod

Un mod minimal ressemble à ça :

// monMod.js
// Toujours commencer par _register pour apparaître dans la liste des mods actifs.

RSC_MOD._register({
  name: 'Mon Super Mod',
  version: '1.0.0',
  author: 'TonPseudo'
});

// Ensuite, tes modifications :
RSC_MOD.addGroup({
  name: 'Les Immortels',
  minPop: 5000,
  bonus: 10,
  members: 8
});

Tu peux encapsuler plusieurs appels dans un IIFE pour éviter de polluer le scope global :

(function() {
  RSC_MOD._register({ name: 'Mod Propre', version: '1.0' });

  // tout ton code ici...
})();

3. API — Joueur

RSC_MOD.setPlayerStat(stat, value)
Modifie directement une statistique du joueur.
statstring'money' | 'fans' | 'popularity' | 'health' | 'addiction' | 'age'
valuenumbernouvelle valeur
RSC_MOD.setSkill(skill, value)
Modifie une compétence du joueur (clampée entre 0 et 100).
skillstring'technique' | 'scene' | 'composition' | 'charisme' | 'marketing' | 'endurance'
valuenumber0 – 100
RSC_MOD.setPlayerStat('money', 100000);
RSC_MOD.setPlayerStat('health', 100);
RSC_MOD.setSkill('technique', 80);
RSC_MOD.setSkill('marketing', 75);

4. API — Groupes

RSC_MOD.addGroup(groupDef) NEW
Ajoute un nouveau groupe dans la liste des groupes rejoignables.
groupDef.namestringNom du groupe (doit être unique)
groupDef.minPopnumberPopularité minimum requise
groupDef.bonusnumberMultiplicateur de gains (ex: 2 = x2)
groupDef.membersnumberNombre de membres (cosmétique)
RSC_MOD.removeGroup(name)
Supprime un groupe existant par son nom.
RSC_MOD.addGroup({
  name: 'Galactic Noise',
  minPop: 4000,
  bonus: 9,
  members: 10
});

RSC_MOD.removeGroup('Les Débutants'); // supprime un groupe existant

5. API — Lieux de concert

RSC_MOD.addVenue(id, venueDef, minPop) NEW
Ajoute ou remplace un lieu de concert.
idstringIdentifiant unique (ex: 'stade'). S'il existe déjà, le lieu est remplacé.
venueDef.namestringNom affiché
venueDef.revenuenumberRevenu de base (€)
venueDef.fansnumberFans de base gagnés
venueDef.difficultynumber0.0 (facile) à 1.0 (impossible) — seuil de skill requis
venueDef.cooldownnumberCooldown en secondes
minPopnumberPopularité requise pour débloquer le lieu
RSC_MOD.patchVenue(id, patch) PATCH
Modifie partiellement un lieu existant. Passe seulement les champs à changer.
// Nouveau lieu ultra-difficile
RSC_MOD.addVenue('stade', {
  name: 'Stade Mondial',
  revenue: 200000,
  fans: 30000,
  difficulty: 0.95,
  cooldown: 120
}, 5000);

// Rendre le bar plus généreux
RSC_MOD.patchVenue('bar', { revenue: 300, cooldown: 8 });

6. API — Albums

RSC_MOD.addAlbumType(albumDef) NEW
Ajoute un nouveau format d'album dans la vue Albums.
albumDef.typestringIdentifiant unique interne (ex: 'mixtape')
albumDef.namestringNom affiché (ex: 'Mixtape')
albumDef.costnumberCoût en €
albumDef.durationnumberCooldown d'enregistrement en secondes
albumDef.minStudionumberNiveau de studio minimum (0–3)
albumDef.tracksnumberNombre de titres (cosmétique)
RSC_MOD.addAlbumType({
  type: 'mixtape',
  name: 'Mixtape Gratuite',
  cost: 0,
  duration: 15,
  minStudio: 0,
  tracks: 7
});
⚠️ Le moteur de calcul de revenus des albums reconnaît les types 'demo', 'ep', 'album', 'live', 'double'. Un type inconnu sera traité comme un 'demo' (revenu unique, pas de passif). Pour des revenus passifs sur un type custom, utilise un hook onTick.

7. API — Équipement

RSC_MOD.addEquipmentCategory(categoryId, label, items) NEW
Crée une nouvelle catégorie complète dans la boutique.
categoryIdstringIdentifiant interne (ex: 'drone')
labelstringTitre affiché dans la boutique
itemsArrayTableau de niveaux : [{ level, name, cost, bonus, desc? }]
RSC_MOD.addEquipmentLevel(categoryId, itemDef) PATCH
Ajoute un niveau supplémentaire à une catégorie existante.
// Nouvelle catégorie : Drones de scène
RSC_MOD.addEquipmentCategory('drone', 'Drones de Scène', [
  { level: 1, name: 'Mini Drone',  cost: 3000,  bonus: 12, desc: 'Effets visuels basiques' },
  { level: 2, name: 'Essaim Drone', cost: 15000, bonus: 35, desc: 'Chorégraphie lumineuse' },
  { level: 3, name: 'Méga Flotte',  cost: 80000, bonus: 70, desc: 'Spectacle unique au monde' }
]);

// Ajouter un niveau 6 à l'instrument
RSC_MOD.addEquipmentLevel('instrument', {
  name: 'Guitare Sacrée',
  cost: 500000,
  bonus: 120,
  desc: 'Forgée dans les flammes du rock'
});

8. API — Achievements

RSC_MOD.addAchievement(achDef) NEW
Ajoute un achievement personnalisé vérifié automatiquement toutes les 5 secondes.
achDef.idstringIdentifiant unique (ex: 'my_mod_ach1')
achDef.namestringNom affiché
achDef.descstringDescription
achDef.iconstringEmoji ou texte court
achDef.checkFunctionFonction sans argument retournant true quand débloqué
achDef.rewardObject{ money?, fans?, popularity?, health? }
RSC_MOD.addAchievement({
  id: 'mod_rich',
  name: 'Méga Riche',
  desc: 'Accumule 1 000 000 €',
  icon: '🤑',
  check: () => player.money >= 1000000,
  reward: { fans: 10000, popularity: 500 }
});

9. API — Événements de concert

RSC_MOD.addConcertEvent(eventDef) NEW
Ajoute un événement aléatoire qui peut se déclencher pendant un concert.
eventDef.chancenumberProbabilité 0.0–1.0 (ex: 0.05 = 5%)
eventDef.typestring'good' | 'bad' | 'neutral' (affecte la couleur du message)
eventDef.namestringNom de l'événement
eventDef.revenueBonusnumberMultiplicateur de revenu (1.0 = inchangé)
eventDef.fanBonusnumberMultiplicateur de fans
eventDef.msgstringMessage affiché au joueur
RSC_MOD.addConcertEvent({
  chance: 0.05,
  type: 'good',
  name: 'Scène Envahie',
  revenueBonus: 1.2,
  fanBonus: 3.0,
  msg: '🎉 Un fan monte sur scène avec toi ! Les fans adorent !'
});

10. API — Réseaux sociaux

RSC_MOD.addSocialPlatform(platformDef) NEW
Ajoute une plateforme sociale dans le menu Réseaux Sociaux.

platformDef.namestringNom affiché
platformDef.iconstringEmoji
platformDef.minPopnumberPopularité requise
platformDef.fanMultipliernumberMultiplicateur de fans par post (ex: 2.5)

11. API — Hooks de jeu

RSC_MOD.onTick(modName, fn) HOOK
Exécute une fonction à chaque seconde de jeu. Utile pour des revenus passifs custom, des malus progressifs, etc.
modNamestringNom du mod (pour les logs console)
fn(gameTime, player)FunctionCallback reçoit gameTime (secondes) et player
RSC_MOD.onConcertEnd(modName, fn) HOOK
Exécute une fonction après chaque concert.
fn(data)FunctionReçoit { type, revenue, fans, success, quality }
💡 Les hooks onTick sont exécutés depuis updateCooldowns(), donc toutes les secondes. Pense à n'agir que toutes les N secondes avec if (gameTime % 60 === 0). 
// Revenus passifs custom toutes les minutes
RSC_MOD.onTick('MonMod', (gameTime, player) => {
  if (gameTime % 60 === 0 && player.skills.marketing >= 80) {
    player.money += 500;
    RSC_MOD.toast('Marketing Passif : +500€');
  }
});

// Bonus après chaque festival réussi
RSC_MOD.onConcertEnd('FestivalBonus', (data) => {
  if (data.type === 'festival' && data.success) {
    player.fans += 500;
    RSC_MOD.toast('Bonus Festival : +500 fans !');
  }
});

12. Exemples complets

🎸 Exemple 1 — Mod de contenu : "Extreme Rock Pack"

(function() {
  RSC_MOD._register({ name: 'Extreme Rock Pack', version: '1.0', author: 'TonPseudo' });

  // Lieu ultime
  RSC_MOD.addVenue('planet', {
    name: 'Concert Planétaire',
    revenue: 1000000,
    fans: 100000,
    difficulty: 0.99,
    cooldown: 300
  }, 9999);

  // Groupe légendaire
  RSC_MOD.addGroup({ name: 'Gods of Rock', minPop: 8000, bonus: 15, members: 12 });

  // Achievement pour le nouveau lieu
  RSC_MOD.addAchievement({
    id: 'erp_planet',
    name: 'Superstar Galactique',
    desc: 'Joue un Concert Planétaire',
    icon: '🌍',
    check: () => (player.festivalPlayed || 0) >= 1 && player.popularity >= 9999,
    reward: { money: 500000, fans: 50000 }
  });
})();

⚙️ Exemple 2 — Mod de rééquilibrage

(function() {
  RSC_MOD._register({ name: 'Easy Mode', version: '1.0' });

  // Rendre tous les concerts 50% plus lucratifs
  ['bar', 'club', 'salle', 'theatre', 'arena', 'festival'].forEach(id => {
    RSC_MOD.patchVenue(id, {
      revenue: Math.floor(venues[id].revenue * 1.5),
      difficulty: Math.max(0.1, venues[id].difficulty - 0.1)
    });
  });

  // Donner un bon départ
  RSC_MOD.setPlayerStat('money', player.money + 5000);
  RSC_MOD.toast('Easy Mode activé !');
})();

13. Référence : objet player

L'objet player est global et accessible directement depuis les mods.

// Propriétés principales
player.name          // string — nom de scène
player.instrument    // string
player.age           // number — âge (décimal)
player.health        // number — 0 à 100
player.addiction     // number — 0 à 100
player.money         // number
player.fans          // number
player.popularity    // number
player.concertsPlayed // number
player.group         // object | null — { name, bonus, members, minPop }
player.albums        // Array d'albums sortis

// Compétences
player.skills.technique   // 0–100
player.skills.scene       // 0–100
player.skills.composition // 0–100
player.skills.charisme    // 0–100
player.skills.marketing   // 0–100
player.skills.endurance   // 0–100

// Équipement (niveau 0 = non acheté)
player.equipment.instrument   // 0–5
player.equipment.amplifier    // 0–4
player.equipment.lights       // 0–4
player.equipment.pyrotechnics // 0–4
player.equipment.soundSystem  // 0–4
player.equipment.transport    // 0–4
player.equipment.studio       // 0–3

// Réseaux sociaux
player.totalPosts       // number
player.fansFromSocial   // number
player.bestPostQuality  // number (0–100)
player.activePlatform   // string — id de la plateforme active

// Achievements débloqués
player.unlockedAchievements // Array de strings (ids)
