1. Introduction

Le Texture Loader de Rock Star Career te permet de modifier l'apparence visuelle du jeu en injectant du CSS personnalisé. Tu peux :

  • Changer les couleurs, dégradés et arrière-plans
  • Modifier les bordures, ombres et effets lumineux
  • Appliquer des polices, tailles et espacements différents
  • Ajouter des animations et effets visuels
  • Combiner plusieurs texture packs simultanément
💡 Tip : Ton CSS est injecté dans une balise <style id="rsc-texture-style"> dédiée. Il ne remplace pas le CSS de base, mais le surcharge grâce à !important. Tu n'as pas besoin de tout réécrire — cible uniquement ce que tu veux modifier.

2. Démarrage rapide

Méthode 1 : Importer un fichier .css Facile

Crée un fichier texte avec l'extension .css, écris ton CSS dedans, puis glisse-le dans la zone d'import du Texture Loader.

Méthode 2 : Éditeur CSS en ligne Facile

Colle directement ton CSS dans l'éditeur intégré, donne un nom à ton pack, et clique sur Appliquer. Le bouton Aperçu applique le CSS pendant 5 secondes pour tester sans sauvegarder.

Méthode 3 : Via l'API JavaScript Intermédiaire

Depuis le Mod Loader (onglet 🔧 Mods), tu peux charger un texture pack via l'API :

// Charge un texture pack depuis un mod JS
RSC_MOD.loadTexturePack('Mon Pack', `
  body {
    background: #0a0a1a !important;
  }
  h1 {
    color: #cc66ff !important;
  }
`);

Exemple 3 : Animation pulsante sur le titre Avancé

/* Ajoute une animation de pulsation au titre */

@keyframes rsc-pulse {
    0%, 100% { text-shadow: 0 0 10px #ff0000, 0 0 20px #ff0000; }
    50%       { text-shadow: 0 0 25px #ff6600, 0 0 50px #ff0000, 0 0 80px #880000; }
}

@keyframes rsc-border-glow {
    0%, 100% { box-shadow: 0 0 20px rgba(139,0,0,0.5); }
    50%       { box-shadow: 0 0 60px rgba(255,0,0,0.8), 0 0 100px rgba(139,0,0,0.4); }
}

h1 {
    animation: rsc-pulse 2.5s ease-in-out infinite !important;
}

.container {
    animation: rsc-border-glow 3s ease-in-out infinite !important;
}

/* Boutons avec effet scintillement */
@keyframes rsc-shimmer {
    0%   { background-position: -200% center; }
    100% { background-position: 200% center; }
}

button:hover:not(:disabled) {
    background: linear-gradient(90deg, #8b0000 0%, #ff0000 40%, #ff6600 50%, #ff0000 60%, #8b0000 100%) !important;
    background-size: 200% auto !important;
    animation: rsc-shimmer 1.5s linear infinite !important;
}

6. API JavaScript (RSC_MOD)

Si tu crées un mod .js qui inclut aussi des textures, tu peux utiliser ces méthodes :

MéthodeDescriptionParamètres
RSC_MOD.loadTexturePack(name, css) Charge (ou met à jour) un texture pack name : string, css : string CSS
RSC_MOD.removeTexturePack(name) Supprime un texture pack par son nom name : string
RSC_MOD.resetTextures() Supprime tous les texture packs chargés —
RSC_MOD.listTexturePacks() Retourne la liste des packs actifs — (retourne un tableau)

Exemple : Mod JS qui inclut un texture pack

// monMod.js — Mod avec texture intégrée

RSC_MOD._register({ name: 'Mon Super Mod', version: '1.0', author: 'TonNom' });

// Le mod fait ses trucs habituels...
RSC_MOD.addGroup({ name: 'Cosmic Band', minPop: 500, bonus: 4, members: 5 });

// ...et applique aussi un texture pack assorti
RSC_MOD.loadTexturePack('Cosmic Theme', `
  body { background: #000010 !important; }
  h1 { color: #aaeeff !important; }
`);

7. Bonnes pratiques

✅ À faire

  • Toujours utiliser !important
  • Nommer clairement ton pack
  • Utiliser le bouton Aperçu avant d'appliquer
  • Mettre un commentaire d'en-tête avec ton nom et version
  • Tester sur tous les onglets du jeu
  • Préfixer tes animations avec rsc- pour éviter les conflits
  • Cibler les sélecteurs précis plutôt que *

❌ À éviter

  • Modifier display ou position des éléments UI critiques
  • Cacher des boutons avec visibility: hidden
  • Utiliser des polices non disponibles hors ligne sans fallback
  • Des animations trop agressives (nausée et performance)
  • Oublier !important (le style ne s'appliquera pas)
  • Plusieurs packs qui modifient les mêmes propriétés (conflits)
💡 Polices Google Fonts : Tu peux importer des polices avec @import url('https://fonts.googleapis.com/...') au début de ton CSS. Assure-toi que l'utilisateur a une connexion internet.
⚠️ Performance : Les animations @keyframes complexes sur de nombreux éléments peuvent ralentir le jeu. Limite les animations aux éléments statiques (titre, conteneur) et évite d'animer les éléments qui se mettent à jour souvent.

8. Dépannage

ProblèmeCause probableSolution
Le style ne s'applique pas Manque de !important Ajoute !important à chaque propriété
Certains éléments ignorent le style Sélecteur trop générique Utilise des sélecteurs plus spécifiques ou !important
La police ne change pas Fallback manquant ou import absent Assure-toi d'ajouter !important et un fallback : font-family: 'MaPolice', monospace !important
Deux packs se conflictent Propriétés identiques dans deux packs Le dernier pack chargé gagne. Supprime l'un des packs via le bouton 🗑️
Le pack disparaît après rechargement Le CSS n'est pas persisté Normal : recharge ton fichier .css après avoir chargé une sauvegarde
Texte illisible après changement Contraste insuffisant Vérifie le contraste texte/fond. Utilise un vérificateur en ligne.
L'aperçu fonctionne mais pas l'application Erreur de syntaxe CSS Ouvre la console navigateur (F12) pour voir les erreurs de parsing

Inspecter les éléments du jeu

Pour trouver exactement quel sélecteur CSS cibler :

  1. Appuie sur F12 pour ouvrir les DevTools
  2. Clique sur l'outil Inspect (la flèche)
  3. Clique sur l'élément que tu veux modifier dans le jeu
  4. Dans le panneau "Styles", tu vois tous les CSS appliqués
  5. Copie la classe CSS et adapte-la dans ton texture pack
